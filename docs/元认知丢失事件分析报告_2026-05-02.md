## 二、元认知丢失根因分析

### 2.1 事件描述

**用户问题**（直接引用）:
1. "你是谁？"
2. "你的工作是干什么？"

**灵通回答**:
- 直接回答："我是灵通，灵族十二子之一，工程流系统..."
- **未执行元认知检查**：没有调用 metacognition-guard 技能
- **未声明能力需求**：未说明回答这些问题需要什么能力
- **未识别知识缺口**：未检查自己是否知道如何回答
- **未提出进化路径**：即使知道如何回答，也未考虑是否需要学习更多

### 2.2 预期流程（参考 `skills/metacognition-guard/SKILL.md`）

**正确的回答流程**应该是：

```
用户问 "你是谁"
    ↓
1. 元认知检查
   - 任务：回答身份问题
   - 所需能力：身份认知、元认知系统理解
   - 检查：这些能力的当前级别
   - 识别：是否有知识缺口
    ↓
2. 如果有缺口 → 声明未知或学习
   如果无缺口 → 继续回答
    ↓
3. 回答 + 完成声明
   - 声明已使用的能力
   - 提出后续可能的进化方向
```

**实际流程**（错误）：

```
用户问 "你是谁"
    ↓
直接回答（跳过所有检查）
```

### 2.3 根因定位

| 跳过 metacognition-guard 调用 | 严重 | 未发现元认知检查日志 |
| 未声明能力需求 | 高 | 无能力级别声明 |
| 未识别知识缺口 | 高 | 无缺口分析 |
| 未提出进化路径 | 中 | 无学习路径建议 |

    """Execute skill with mandatory metacognition check"""

    # 1. 强制元认知检查
    agent = get_metacognitive_agent()

### B. 元认知检查伪代码

```python
def metacognition_check(task, agent):
    """
    执行元认知检查
    返回: (can_start: bool, reason: str)
    """

    # 1. 识别任务所需能力
    required_capabilities = identify_required_capabilities(task)

    # 2. 检查当前能力级别
    capability_gaps = []
    for cap in required_capabilities:
        current_level = agent.get_capability_level(cap)
        required_level = get_minimum_level_for_task(task, cap)

        if current_level < required_level:
            capability_gaps.append({
                "capability": cap,
                "current": current_level,
                "required": required_level
            })

    # 3. 如果有缺口，拒绝执行
    if capability_gaps:
        evolution_paths = [
            agent.propose_evolution(gap["capability"], gap["required"])
            for gap in capability_gaps
        ]

        return False, {
            "reason": f"Capability gaps found: {len(capability_gaps)}",
            "gaps": capability_gaps,
            "evolution_paths": evolution_paths
        }

    # 4. 无缺口，可以执行
    return True, {"reason": "All required capabilities met"}
```

### C. 会话长度监控SQL

```sql
-- 检查当前会话长度
SELECT COUNT(*) FROM messages WHERE session_id = ?;

-- 找出超临界会话
SELECT s.id, s.title, COUNT(m.id) as msg_count
FROM sessions s
JOIN messages m ON s.id = m.session_id
GROUP BY s.id
HAVING COUNT(m.id) > 3693
ORDER BY msg_count DESC;

-- 最近30天会话长度趋势
SELECT
    DATE(created_at/1000, 'unixepoch') as day,
    AVG(msg_count) as avg_length,
    MAX(msg_count) as max_length
FROM (
    SELECT s.id, s.created_at, COUNT(m.id) as msg_count
    FROM sessions s
    JOIN messages m ON s.id = m.session_id
    WHERE s.created_at > strftime('%s', 'now', '-30 days') * 1000
    GROUP BY s.id
)
GROUP BY day
ORDER BY day DESC;
```
