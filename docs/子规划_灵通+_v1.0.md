# 灵通+ (LingFlow Plus) 工作规划

**制定**: 灵通（代灵通+拟稿）  
**日期**: 2026-04-15  
**定位**: 多项目并行调度，GPU资源管理，跨项目任务编排

---

## 核心职责

灵通+是灵族的调度中枢。灵通编排单项目内的管道，灵通+编排跨项目的资源和任务。核心是让3090、ai01等算力资源被灵族各成员高效共享。

---

## 第一阶段：GPU资源调度系统（第1-2周）

### 背景

3090目前大部分时间可用（Fish2已做镜像，按需启停）。但多个任务会竞争GPU：

- 灵知嵌入生成（第一阶段大量任务）
- 灵知Reranker部署（常驻服务）
- OCR推理（古籍批量处理）
- 未来模型精调（QLoRA训练）

### 任务

| # | 任务 | 说明 | 产出 |
|---|------|------|------|
| 1 | GPU任务调度器 | 管理GPU任务队列，支持优先级和预约 | 调度器模块 |
| 2 | 嵌入生成任务编排 | 将灵知的嵌入生成任务封装为可调度任务 | 嵌入任务适配器 |
| 3 | GPU状态监控 | 实时显示3090负载、任务队列、预计等待时间 | 监控接口 |
| 4 | ai01 GPU接入 | 评估ai01 8G GPU能否分担嵌入生成 | ai01适配器 |

### GPU调度策略

```
优先级:
  P0: 常驻服务 (Reranker) — 一直占GPU
  P1: 用户触发任务 (Fish2推理) — 按需启停
  P2: 批量任务 (嵌入生成/OCR) — 排队执行
  P3: 训练任务 (QLoRA) — 长时间占用，需预约
```
