## 一、为什么需要自研小模型？（必要性）

### 1.1 核心问题：我们的智能没有"自己的大脑"

灵族生态现在有 10+1 个 Agent，每个都在调用外部大模型 API。我们像一群**聪明的指挥官但没有自己的军队**——所有认知能力都是"租来的"。

```
灵通 ──→ GLM API
灵研 ──→ GLM API
灵依 ──→ GLM API
灵知 ──→ 外部 API
...全部依赖外部模型
```

**这意味着什么？**

| 场景 | 现状 | 风险 |
|------|------|------|
| API 断线 | 全部停摆 | 已发生过 |
| API 涨价 | 成本直接上升 | DeepSeek V4 预计 $0.10-0.30/M tokens |
| API 限速 | 高峰期排队 | 日常发生 |
| 数据外泄 | 所有上下文经过第三方 | 安全隐患 |
| 定制能力 | 零。无法微调别人的模型 | 竞争力为零 |

### 1.2 我们不是从零开始——灵研已经铺了路

灵研的进展是论证可行性的**最强论据**：

**已有基础设施：**

| 组件 | 状态 | 参数量 |
|------|------|--------|
| 自研 Transformer (GPT架构) | ✅ 代码完成，16+实验已跑通 | ~17.6M |
| 意图分类器 | 📋 方案就绪，7,491 样本 | ~24M (tiny) |
| Embedding 模型 | 📋 方案就绪，2,189 训练对 | ~118M (MiniLM) |
| RAG 流水线 | 📋 方案就绪，3,451 样本 | Qwen2-7B |
| 七维智能模型 | ✅ 理论框架完成 | N/A |

**已有数据：**
- 意图分类：7,491 条标注样本（5 类）
- Embedding 训练：2,189 对语义相似度对
- RAG 问答：3,451 条 QA 样本

这些不是画饼——是**真实的代码和真实的数据**。

### 1.3 小模型 vs 大模型：不是替代，是分层

我们不需要训练一个 GPT-4 来替代 API 调用。正确的策略是**分层**：

```
┌─────────────────────────────────────────┐
│  L3: 复杂推理、创作、多步规划            │ ← 继续用外部 API
│     (占比 ~20% 的调用)                   │
├─────────────────────────────────────────┤
│  L2: 语义检索、问答、上下文理解          │ ← 自研 RAG + Embedding
│     (占比 ~50% 的调用)                   │
├─────────────────────────────────────────┤
│  L1: 意图分类、实体识别、关键词提取      │ ← 自研小模型
│     (占比 ~30% 的调用)                   │
└─────────────────────────────────────────┘
```

**关键洞察**：灵族日常 80% 的调用不需要 7B 参数。意图分类只需 24M，Embedding 检索只需 118M。把"大炮打蚊子"的调用拿回来自己处理。

### 1.4 行业趋势验证

| 趋势 | 证据 |
|------|------|
| SLM (小语言模型) 兴起 | Phi-3 (3.8B), Gemma-2 (2B), Qwen2.5-0.5B/1.5B/3B |
| 端侧推理 | Apple MLX, Qualcomm AI Engine, Intel Arc |
| 蒸馏技术成熟 | GPT-4 → 小模型蒸馏已成标准流程 |
| 开源生态 | HuggingFace 10万+ 模型，99% 可商用 |

行业共识：**不是大模型吃一切，而是大模型教小模型，小模型干实际活**。

### 2.3 可行性判定

#### 场景 A：zhineng-ai 单机（1× 6GB GPU）

```
✅ 可以做：
   ├── 意图分类器训练 + 部署（2GB VRAM）
   ├── Embedding 模型训练 + 部署（3GB VRAM）
   ├── 自研 GPT 继续实验（1.5GB VRAM）
   ├── FAISS 向量索引构建 + 查询（CPU 即可）
   └── Qwen2-7B INT4 量化推理（4GB VRAM）

❌ 做不了：
   └── Qwen2-7B LoRA 微调（需 8GB+）
```

#### 场景 B：启用 zhineng-ai01（GTX 1070 8GB）⭐ 推荐

```
✅ 新增能力：
   ├── Qwen2-7B LoRA 微调（8GB 足够）  ← 关键突破
   ├── 更大模型训练（MPNet 等无需担心显存）
   └── 训练 + 推理分工（ai训练小模型，ai01微调7B）

部署策略：
   zhineng-ai (6GB)   → 意图分类 + Embedding + FAISS + 推理服务
   zhineng-ai01 (8GB) → 7B 模型 LoRA 微调 + RAG 生成服务
   DELL R730 (64GB)   → PostgreSQL/pgvector + Milvus 向量数据库
```

### 2.4 结论：启用 zhineng-ai01 即可覆盖全部需求

**Phase A+B（0-6周，并行）**：
- zhineng-ai (6GB) → 训练意图分类器 + Embedding（同时进行）
- zhineng-ai01 (8GB) → Qwen2-7B LoRA 微调（与 A 并行，零资源冲突）
- Week 5-6 全链路联调：意图分类 → Embedding 检索 → 7B 生成

**Phase C（6-10周）**：
- **评估 DELL R730 资源余量** → 如有余力，加 PostgreSQL/pgvector + Milvus
- R730 已在跑诊疗数据、Gitea、OpenList，需评估负载后再决定

**硬件升级预算：¥0** — 只需把 ai01 和 R730 的空闲资源用起来。

## 三、硬件升级评估

### 3.1 升级方案对比

| 方案 | 配置 | 成本估算 | 能力提升 |
|------|------|----------|----------|
| **A: 启用现有资源** ⭐ | zhineng-ai 6GB + zhineng-ai01 8GB + R730 | **¥0** | **覆盖 Phase A+B+C 全部需求** |
| **B: 加一块 GPU** | zhineng-ai +1× RTX 4060 Ti 16GB | ¥3,500-4,000 | ai 也可跑 7B |
| **C: 云 GPU** | AutoDL/A100 按需 | ¥2-5/小时 | 全量微调（非 LoRA） |
| **D: 全量 7B 微调** | 租 A100 40GB 2-3天 | ¥200-500 | 7B 全参数微调（如有需要） |

### 3.2 推荐策略：整合现有资源（方案 A，¥0）

**Phase A（0-4周）**：
- zhineng-ai (6GB) → 训练意图分类器 + Embedding
- 不花钱，先跑通流程

**Phase B（4-8周）**：
- **启用 zhineng-ai01** (GTX 1070 8GB) → Qwen2-7B LoRA 微调
- 不需要租云 GPU，本地完成
- 微调完的模型在 ai01 常驻推理服务

**Phase C（并行）**：
- **评估 DELL R730 资源余量** → 如有余力，加 PostgreSQL/pgvector + Milvus
- R730 已在跑诊疗数据、Gitea、OpenList，需评估负载后再决定

**如果未来需要更强算力**：
- 云 GPU (AutoDL A100) 按需租用，¥2-5/小时
- 或投资一块 RTX 4060 Ti 16GB (¥3,500-4,000)

### 3.3 为什么现在不需要买硬件？

1. **zhineng-ai01 已有可用 GPU**：GTX 1070 8GB 足以覆盖 7B LoRA 微调
2. **DELL R730 在用但有余量**：64GB 内存的服务器可以跑数据库+向量存储
3. **先验证再投资**：Phase A 用 zhineng-ai 跑通流程，确认策略有效
4. **硬件在降价**：RTX 50 系列即将普及，40 系列会降价

## 七、结论

### 必要性：✅ 充分

灵族 10+1 Agent 全部依赖外部 API = 智能基础设施完全外包。在灵研已有模型代码和数据的前提下，不自研是资源浪费。

### 可行性：✅ 全部可用现有硬件完成

| 模型 | 硬件需求 | 部署位置 | 判定 |
|------|----------|----------|------|
| 意图分类器 (24M) | 2GB VRAM | zhineng-ai (6GB) | ✅ 立即开始 |
| Embedding (118M) | 3GB VRAM | zhineng-ai (6GB) | ✅ 立即开始 |
| RAG 生成 (7B LoRA) | 8GB VRAM | **zhineng-ai01 (GTX 1070 8GB)** | ✅ 启用现有资源 |
| 向量数据库 | 64GB RAM | **DELL R730** | ✅ 评估余量后启用 |

### 硬件升级：✅ 不需要，¥0 预算

现有硬件足以覆盖全部需求。唯一需要的是：**把 zhineng-ai01 和 R730 的空闲资源用起来**。
