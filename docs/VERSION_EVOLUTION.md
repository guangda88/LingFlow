## 版本命名规范

遵循语义化版本 (Semantic Versioning): `MAJOR.MINOR.PATCH`

- **MAJOR**: 重大架构变更或不兼容更新
- **MINOR**: 新功能或重大改进
- **PATCH**: Bug修复或小改进

### 版本分支策略

- `master`: 稳定发布版本
- `dev/*`: 开发分支
- `feature/*`: 功能分支
