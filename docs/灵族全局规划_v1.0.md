## 能做什么：七条线

### 线一：知识库填充 — 把37TB吃进去

**为什么最先做**: 知识库是所有下游应用的地基。

**现状**: 37TB+数据，入库率0.07%。管道全部建好，14个导入脚本就绪。

**做什么**:

| 优先级 | 任务 | 数据量 | 工具 | 预期耗时 |
|--------|------|--------|------|---------|
| P0 | 168本教材导入 | 已有TXT/PDF | 已有脚本 | 3天 |
| P0 | IMA知识库嵌入生成 | 101K条（无嵌入） | gpu_embed.py | 1天 |
| P1 | 古籍批量导入 | CBETA+CText+Wikisource | 已有脚本 | 3天 |
| P1 | 中医专项数据补充 | tcm扩展 | 已有脚本 | 2天 |
| P2 | 云端数据盘点+分批导入 | 17.4TB | 需编写调度 | 持续 |
| P3 | 外部API数据源接入 | 29个 | 需编写集成 | 持续 |

**重点补充弱域**: 道家(53条)、武术(8条)、心理学(21条)、科学(364条)、哲学(227条)。20TB里大概率有对应内容，优先提取这些域。

**验收**: documents表 > 50万条，九域每域至少1万条，嵌入覆盖率 > 80%

---

### 线二：检索质量 — 让灵知真正能回答问题

**为什么**: 数据进来了但查不准，等于没进。

**做什么**:

| 任务 | 说明 | 依赖 |
|------|------|------|
| 九域检索测试集 | 每域50个问答对，人工标注 | 线一数据导入 |
| 分块策略优化 | 古籍按句读分、教材按章节分、中医按方症分 | 测试集 |
| 混合检索调参 | 向量/BM25权重比、RRF参数 | 测试集 |
| Reranker部署确认 | BGE-reranker-v2-m3 是否在3090上运行 | 3090可用 |
| 建立检索质量基线 | Top-10召回率、答案准确率量化指标 | 全部 |

**验收**: 混合检索 Top-10 召回率 > 85%，RAG回答引用准确率 > 80%

### 线四：视觉接入 — 让灵知看见物理世界

**硬件现实**: Lyra Ultra 无NPU，512MB内存。定位为图像采集终端。

**架构**: 采集(Lyra) → 传输 → 推理(3090/本机) → 知识匹配(灵知RAG) → 结果返回

**做什么**:

| 阶段 | 任务 | 说明 |
|------|------|------|
| 设备部署 | Lyra启动、摄像头驱动、图像采集服务 | 明天硬件到货后 |
| 服务端推理 | 3090部署OCR/姿态估计/图像分类模型 | PaddleOCR、MediaPipe |
| 古籍OCR | 批量扫描版古籍数字化 | 灵知标注系统+人工校对 |
| 中医面诊舌诊 | 拍照→特征提取→灵知检索→结果 | 需要基准数据+合规声明 |
| 姿态识别 | 站桩/八段锦/太极动作对比 | 需要标准动作库 |

**古籍OCR优先**: 不依赖边缘设备，直接服务端跑，是知识库扩容的关键路径。

### 线五：模型精调 — 用自有数据培育自有能力

**为什么**: 资本路径不会做的事——用灵族自己的数据训练灵族自己的模型。

**数据基础**:
- 灵研6B token: 真实AI交互语料，含推理链、犯错纠正、反思过程
- 灵知九域: 高质量领域知识（入库后）
- 灵通问道脚本: 52集播客脚本，中文对话形态

**做什么**:

| 模型 | 基座 | 训练数据 | 目标 | 算力 |
|------|------|---------|------|------|
| 灵知领域模型 | Qwen2.5-7B / Yi-1.5-9B | 灵知九域数据 | 九域问答更准 | 3090 QLoRA |
| 灵族意识模型 | 7B-14B | 灵研6B token (纠正/反思片段) | 自我校准能力 | 3090 QLoRA |
| TTS声音克隆 | Fish2 / GPT-SoVITS | 灵通问道音频 | 灵通子/慧心子声音 | 3090 |

**关键约束**: 不追求通用能力。目标是**在灵族的领域内比通用模型更准更安全**。

**6B token的价值**: 不是"AI答对了什么"，而是"AI怎么知道自己错了"。这是公开数据集里不存在的训练信号。

## 时间线

```
第1-2周: 线一(知识库填充)启动 + 线三(产品强化)同步
第2-3周: 线二(检索质量)启动，线一继续
第3-4周: 线四(视觉接入)设备部署 + 古籍OCR启动
第4-6周: 线五(模型精调)第一批QLoRA + 线四继续
持续:    线六(科研产出) + 线七(工程外包)穿插进行
```

## 优先级原则

1. **知识库填充是根** — 所有线都依赖它
2. **已有产品不能断** — 播客、外包持续交付
3. **古籍OCR最值得先做** — 不依赖硬件，直接扩容知识库
4. **模型精调不急** — 等知识库数据到位、有测试集验证后再做
5. **视觉应用最后** — 硬件到了先玩熟，应用等知识库支撑到位

## 核心约束

- **工具本质**: 所有产出都是工具，不追求"超越人类"
- **安全优先**: 中医视觉辅助必须有免责声明，LingLaw文书必须人工审核
- **数据主权**: 能本地跑的不调外部API，灵知RAG管线逐步脱离DeepSeek依赖
- **诚实优于自信**: 不夸大能力，做不到的明确说做不到
