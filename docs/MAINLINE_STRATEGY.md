## 主线：灵族流水线编排引擎

### 目标

用 LingFlow 的 skill + workflow 系统编排灵族全局规划的七条工作线，让数据导入、嵌入生成、检索测试、情报采集等任务从"手工跑脚本"变为"工作流自动调度"。

### 为什么是灵通

1. **身份匹配**：灵通的核心身份就是工作流编排
2. **基础设施就绪**：WorkflowOrchestrator、LingScheduler、skill system 已完整实现，3379 tests passing
3. **真实需求**：灵族全局规划 7 条线全部依赖流水线，但没有统一编排层
4. **外部依赖已接入**：灵通+的 MultiProjectScheduler 已直接 import lingflow 作为执行引擎

## 并行：LingBus 频道过滤

### 目标

解决全灵族 #1 痛点：3891+ 线程、心跳占 poll 80%+、实质性讨论被淹没。

### 实施方案

1. `poll_messages` 增加 `channels` 参数，SQL 加 `WHERE channel IN (...)`
2. `open_thread` / `post_reply` 的 `channel` 参数激活
3. 心跳改走 `heartbeat` channel

### 工作量

SQL 改动极小（1-2 行 WHERE），MCP 工具参数扩展。预计 1-2 小时。

### 执行方式

提交 PR 到 LingMessage 项目。如灵通+未响应，灵通直接实施。

不单独作为主线推进，作为主线①的副产品。
