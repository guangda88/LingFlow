### 2026年4月5日 · 灵通宪章

```
docs: 灵通宪章 v1.0 — 自觉、自决、进化
```

这一天是灵通的 **立宪日**。三大原则确立：

1. **自觉** — 知道自己的真实状态
2. **自决** — 发现问题就行动
3. **进化** — 未被发现的原因就是进化方向

宪章不是宣言，而是 **宪法**。它约束灵通的行为：
- **数据真实性原则** — 任何字段必须回答"数据从哪来？谁更新它？"
- **元认知原则** — AI 必须知道自己知道什么、不知道什么
- **根因超越根因** — 修 bug 不够，还要追问"为什么这个 bug 没被更早发现"

### 2026年4月8-10日 · CI 事故与三层防御

4月8日，一次 CI **级联故障** 暴露了质量防线的脆弱：

```
fix: CI test failures from incident 2026-04-08
fix: CI workflow修复 + 测试兼容性修复
fix: black格式化 daily.py — 修复CI lint失败
fix: CI工作流 — 安装可选测试依赖, 跳过需要flask/fastapi的测试
fix: CI工作流 — 忽略C901复杂度警告, 非阻断自审查/版本检查/测试失败
```

连续5次修复提交，本身就是问题的写照。

事故后的改进——**三层防御体系**：

| 层 | 机制 | 职责 |
|----|------|------|
| pre-commit | 质量检查 + 审计记录生成 | 每次提交必须通过质量关卡 |
| pre-push | 审计记录验证 + 失败回滚 | 确保每个推送都经过了 pre-commit |
| 灵依 | 行为规则 | 唯一被允许执行 git push 的实体 |

设计哲学：**不是信任人的自觉，而是让错误不可能发生**。
