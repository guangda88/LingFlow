## 核心类和函数

### 1. AgentCoordinator

**核心职责:**
- 任务调度和执行
- 代理选择和分配
- 状态监控和管理

**核心方法:**
```python
submit_task(task)           # 提交任务
schedule_tasks()            # 调度任务
execute_parallel(tasks)      # 并行执行
execute_workflow(tasks)      # 执行工作流
get_status()                # 获取状态
```

### 2. AgentRegistry

**核心职责:**
- 代理注册和发现
- 能力匹配查询
- 代理状态管理

**核心方法:**
```python
register_agent(agent)       # 注册代理
find_agents_for_task(task)  # 查找代理
get_agent(name)            # 获取代理
list_agents()              # 列出代理
```

### 3. ContextCompressor

**核心职责:**
- 上下文压缩
- 信息密度分析
- 压缩统计

**核心方法:**
```python
compress(context)          # 压缩上下文
estimate_tokens(text)     # 估算 token
get_stats()                # 获取统计
```

## 数据流

### 任务执行数据流

```
Task (定义)
  ↓
Task + Context (压缩后)
  ↓
Agent (执行)
  ↓
TaskResult (结果)
  ↓
CompletedTasks (存储)
```

### 并行执行数据流

```
Task[] (多个任务)
  ↓
ScheduleTasks[] (已调度)
  ↓
ExecuteParallel[] (并行执行)
  ↓
TaskResult[] (多个结果)
  ↓
Results (聚合结果)
```

## 错误处理流程

### 任务执行失败

```
Task.execute() 失败
  ↓
捕获异常
  ↓
返回 TaskResult(success=False, error=...)
  ↓
记录到 failed_tasks
  ↓
继续执行其他任务
```

### 无可用代理

```
find_agents_for_task() 返回空列表
  ↓
返回 TaskResult(success=False, error="No suitable agent")
  ↓
记录到 failed_tasks
  ↓
跳过该任务
```

### 依赖未满足

```
schedule_tasks() 发现依赖未满足
  ↓
跳过该任务
  ↓
等待依赖完成
  ↓
下一轮调度重新尝试
```

## 总结

lingflow v1.1.0 的核心业务流程围绕三个主要组件：

1. **AgentCoordinator**: 任务调度和执行
2. **AgentRegistry**: 代理管理和发现
3. **ContextCompressor**: 上下文优化

通过这些组件的协同工作，实现了高效、可靠的多代理任务执行系统。
