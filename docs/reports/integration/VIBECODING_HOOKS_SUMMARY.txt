========================================
lingflow Hooks v2.0 部署总结
========================================

部署时间: 2026-03-29
部署方法: VibeCoding 渐进式部署
状态: ✅ 成功

========================================
核心原则
========================================

1. 质量门控
   - P0: 基础质量（必需，可能阻断）
   - P1: 文档规范（警告，不阻断）
   - P2: 增强功能（建议，不阻断）

2. AI 辅助
   - 自动检测代码问题
   - 提供智能修复建议
   - 降低修复门槛

3. 渐进式实施
   - 从低级别开始
   - 根据团队适应情况提升
   - 支持快速迭代模式

========================================
默认配置
========================================

P0 检查（基础质量）:
  flake8    : 阻断 (2) - 代码质量检查
  tests     : 警告 (1) - 测试状态

P1 检查（文档规范）:
  docs      : 警告 (1) - 文档完整性
  format    : 警告 (1) - 代码格式
  commit_msg: 阻断 (2) - 提交消息格式

P2 检查（增强功能）:
  perf      : 关闭 (0) - 性能基准
  multi_repo: 关闭 (0) - 多仓库一致性
  complexity: 关闭 (0) - 代码复杂度

========================================
使用方式
========================================

日常开发:
  git commit -m "feat(agent): 添加功能"

快速迭代模式:
  HOOKS_FAST_ITERATION=1 git commit -m "feat: 快速迭代"

自定义配置:
  HOOKS_P0_TESTS=0 HOOKS_P1_DOCS=0 git commit

跳过检查:
  git commit --no-verify -m "message"

========================================
提交消息格式
========================================

<type>(<scope>): <subject>

类型: feat, fix, docs, style, refactor, test, chore, perf, ci
作用域: agent, workflow, coordination, monitoring, core, utils

示例:
  feat(agent): 添加新的代码审查 Agent
  fix(workflow): 修复并行执行死锁
