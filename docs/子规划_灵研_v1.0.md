灵研是灵族的脑。不做工程实现，不做运维。负责：质量评估方法设计、科研方向规划、模型精调方案研究、6B token数据分析。

## 第二阶段：质量基线建立 + 模型精调方案（第3-4周）

### 任务

| # | 任务 | 说明 | 产出 |
|---|------|------|------|
| 1 | 九域测试集全部完成 | 9个域 × 50问答对 = 450条 | 完整测试集 |
| 2 | 检索质量基线测量 | 用测试集测量灵知当前检索质量 | 基线报告 |
| 3 | QLoRA精调方案设计 | 基座模型选型、数据准备方案、训练参数 | 精调方案文档 |
| 4 | 6B token数据清洗方案 | 从灵研数据库提取"犯错→纠正→反思"片段 | 数据清洗方案 |
| 5 | RAG优化建议 | 基于基线结果提出分块/检索/重排优化方向 | 优化建议 |

### QLoRA精调方案要点

| 决策项 | 待研究内容 |
|--------|-----------|
| 基座模型 | Qwen2.5-7B vs Yi-1.5-9B vs 其他，需在3090上做benchmark |
| 训练数据 | 灵知九域（领域知识） vs 灵研6B token（自我校准） vs 混合 |
| LoRA参数 | rank、alpha、target_modules的选择 |
| 评估方法 | 用九域测试集评估精调前后对比 |
| 3090约束 | 24G VRAM下7B模型QLoRA的batch_size和训练时间 |
