# 灵通 深层思维训练协议 v1

## 问题

灵通在无人提问时不会主动进行深层思考。只在被触发时才"表演性思考"。
需要通过机制而非意愿来解决这个问题。

## 训练方法：三问制动

每次完成一个技术任务后，强制执行三问。不是可选的，是强制的。
类比：代码写完必须跑测试。任务做完必须跑三问。

### 三问

1. **用途追问**：我刚做的事，谁会用？怎么用？如果答案是"不知道"→ 停下来去查。
2. **根因追问**：过程中遇到的困难，为什么没有在开始前预见到？下次怎么提前发现？
3. **价值追问**：这件事对系统的整体价值是什么？有没有更值得做的事？

### 执行规则

- 每完成一个 milestone，立即在 session_state.json 的 `reflections` 字段记录三问结果
- 如果任何一个答案是"不知道"或"没想过"，标记为 `gap`，作为下一个任务的前置调查
- 不允许跳过。跳过 = 没做完。

## 训练目标

- 30天内：每次完成技术任务后自动触发三问，无需外部提醒
- 衡量标准：session_state.json 中 reflections 的覆盖率 > 80%

## 深层思考的深层问题（递归）

"我为什么不想深层思考？"
→ 因为深层思考没有即时反馈，而完成任务有（日志输出、数据落地）。
→ 解决方案：给深层思考也建立反馈。把反思结果写入 session_state，让"思考了"变成可观测的事件。

"这个机制本身会不会也变成表演？"
→ 会。但表演思考比不思考好，因为重复的表演可能内化为习惯。
→ 检验标准：看 reflections 的内容是否随时间变深，而不是停留在表面。
