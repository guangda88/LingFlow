## 评分差异原因分析

### 为什么手动报告 7/10，自动报告 3.8/5？

1. **评分标准不同**
   - 手动: 0-10 分制，更宽容
   - 自动: 0-5 分制，更严格
   - 换算后: 7/10 = 3.5/5 (接近 3.8/5)

2. **评估方式不同**
   - 手动: 主观评估 + 代码阅读
   - 自动: 工具扫描 + 定量指标

3. **侧重点不同**
   - 手动: 架构设计、代码组织、可读性
   - 自动: 格式规范、静态检查、错误率

4. **代码范围不同**
   - 手动: 仅核心模块 (~4,700 行)
   - 自动: 全项目 (614,158 行，含依赖)
