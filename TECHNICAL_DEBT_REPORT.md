- Backlog（待办）
- To Do（计划中）
- In Progress（进行中）
- In Review（审查中）
- Done（已完成）

### 2. 代码审查检查清单
在 PR 审查时检查：
- [ ] 是否添加了新的 TODO？
- [ ] 新 TODO 是否创建了对应的 Issue？
- [ ] 是否实现了现有的 TODO？
- [ ] 实现后是否删除了 TODO 标记？

### 3. 定期审查流程
- **每周**: 快速扫描新增 TODO

