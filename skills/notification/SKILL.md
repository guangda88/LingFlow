# notification 技能

## 技能概述

notification 是一个用于发送通知的技能，它可以向用户发送各种类型的通知，包括信息、警告和错误通知。

## 功能特性

- **信息通知**：发送一般信息通知
- **警告通知**：发送警告通知
- **错误通知**：发送错误通知
- **自定义通知**：发送自定义类型的通知
- **多渠道通知**：支持多种通知渠道

## 使用场景

- 当你需要在工作流中发送通知时
- 当你需要向用户传达信息时
- 当你需要在任务完成或失败时发送通知时
- 当你需要批量发送通知时

- 无直接依赖关系

## 使用方法

### 1. 发送信息通知

```bash
# 使用命令行发送信息通知
lingflow run notification --params '{"message": "任务执行成功", "level": "info"}'
```

### 2. 发送警告通知

```bash
# 发送警告通知
lingflow run notification --params '{"message": "检测到潜在问题", "level": "warning"}'
```

### 3. 发送错误通知

```bash
# 发送错误通知
lingflow run notification --params '{"message": "任务执行失败", "level": "error"}'
```

## 最佳实践

1. **及时通知**：在任务完成或失败时及时发送通知
2. **清晰明确**：通知内容应该清晰明确，便于用户理解
3. **适当频率**：避免发送过多的通知，以免打扰用户
4. **分类通知**：根据通知的重要性和类型选择合适的通知级别
5. **集成工作流**：将通知集成到工作流中，实现自动化通知

## 故障排除

- **通知发送失败**：检查通知配置是否正确，以及是否有足够的权限发送通知
- **通知未收到**：检查通知渠道是否正常，以及用户是否正确配置了通知接收方式
- **通知内容错误**：检查通知内容是否正确，以及是否包含必要的信息
- **通知频率过高**：调整通知频率，避免发送过多的通知
