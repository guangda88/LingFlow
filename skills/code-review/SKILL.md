# code-review 技能

## 技能概述

code-review 是一个用于代码审查的技能，它可以分析代码质量、识别问题并提供改进建议。

### 8维代码审查框架

本技能基于8个关键维度进行全面的代码审查：

1. **代码质量** - 代码规范、可读性、复杂度
2. **架构设计** - 模块化、可扩展性、设计模式
3. **性能分析** - 性能瓶颈、内存优化、效率提升
4. **安全性** - 安全漏洞、依赖风险、数据保护
5. **可维护性** - 代码结构、文档、测试覆盖
6. **最佳实践** - 编程规范、设计原则、行业标准
7. **autoresearch理念一致性** - 符合核心理念的程度
8. **潜在Bug分析** - 运行时错误、数据完整性、边界条件

## 功能特性

### 8维审查功能

**1. 代码质量分析**：
- 分析代码的质量和可维护性
- 识别代码复杂度和可读性问题
- 检查代码风格是否符合规范

**2. 架构设计审查**：
- 评估模块化和可扩展性
- 检查设计模式的使用
- 分析组件间的耦合度

**3. 性能分析**：
- 识别性能瓶颈
- 分析内存使用和优化空间
- 提供性能提升建议

**4. 安全检查**：
- 检测潜在的安全漏洞
- 分析依赖项的安全性
- 评估数据保护措施

**5. 可维护性评估**：
- 评估代码结构和组织
- 检查文档完整性
- 分析测试覆盖率

**6. 最佳实践遵循**：
- 检查Python/ML最佳实践
- 验证设计原则应用
- 对照行业标准

**7. autoresearch理念一致性**：
- 验证符合核心理念程度
- 检查关键约束遵循
- 评估设计一致性

**8. 潜在Bug分析**：
- 识别运行时错误风险
- 分析数据完整性问题
- 检查边界条件和异常处理

- **改进建议**：提供具体的改进建议
- **代码风格检查**：检查代码风格是否符合规范

## 使用场景

- 当你需要全面审查代码时（8维度）
- 当需要特定维度审查时（代码质量、安全性等）
- 当你需要识别代码中的问题时
- 当你需要性能优化建议时
- 当你需要在工作流中集成代码审查功能时
- 当你需要批量审查代码时
- 当需要评估autoresearch理念一致性时

## 使用方法

### 1. 审查特定文件

```bash
# 使用命令行审查特定文件
lingflow run code-review --params '{"focus": "duplicate_code", "files": ["file1.py", "file2.py"]}'
```

### 2. 审查整个目录

```bash
# 审查整个目录
lingflow run code-review --params '{"focus": "code_quality", "files": ["./lingflow/"]}'
```

## 最佳实践

1. **定期审查**：定期审查代码，确保代码质量
2. **全面审查**：从多个角度审查代码，包括质量、安全性和性能
3. **反馈机制**：建立有效的反馈机制，及时解决审查发现的问题
4. **自动化集成**：将代码审查集成到工作流中，实现自动化审查
5. **持续改进**：根据审查结果持续改进代码质量

## 故障排除

- **审查失败**：检查输入参数是否正确，以及是否有足够的权限访问文件
- **分析错误**：检查代码是否存在语法错误或其他问题
- **性能问题**：对于大型代码库，可能需要增加审查时间或分批审查
- **结果不准确**：检查审查配置是否正确，以及是否使用了最新的审查规则
