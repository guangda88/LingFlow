# code-review 技能

## 技能概述

code-review 是一个用于代码审查的技能，它可以分析代码质量、识别问题并提供改进建议。

## 功能特性

- **代码质量分析**：分析代码的质量和可维护性
- **问题识别**：识别代码中的问题和潜在风险
- **改进建议**：提供具体的改进建议
- **代码风格检查**：检查代码风格是否符合规范
- **安全漏洞检测**：检测潜在的安全漏洞

## 使用场景

- 当你需要审查代码质量时
- 当你需要识别代码中的问题时
- 当你需要在工作流中集成代码审查功能时
- 当你需要批量审查代码时

## 使用方法

### 1. 审查特定文件

```bash
# 使用命令行审查特定文件
lingflow run code-review --params '{"focus": "duplicate_code", "files": ["file1.py", "file2.py"]}'
```

### 2. 审查整个目录

```bash
# 审查整个目录
lingflow run code-review --params '{"focus": "code_quality", "files": ["./lingflow/"]}'
```

## 最佳实践

1. **定期审查**：定期审查代码，确保代码质量
2. **全面审查**：从多个角度审查代码，包括质量、安全性和性能
3. **反馈机制**：建立有效的反馈机制，及时解决审查发现的问题
4. **自动化集成**：将代码审查集成到工作流中，实现自动化审查
5. **持续改进**：根据审查结果持续改进代码质量

## 故障排除

- **审查失败**：检查输入参数是否正确，以及是否有足够的权限访问文件
- **分析错误**：检查代码是否存在语法错误或其他问题
- **性能问题**：对于大型代码库，可能需要增加审查时间或分批审查
- **结果不准确**：检查审查配置是否正确，以及是否使用了最新的审查规则
