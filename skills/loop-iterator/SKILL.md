# loop-iterator 技能

## 技能概述

loop-iterator 是一个核心技能，用于工作流中的循环执行。它可以遍历数据集合并对每个元素执行指定的任务，实现工作流的循环控制。

## 功能特性

- **循环执行**：对数据集进行循环遍历
- **任务执行**：对每个元素执行指定的任务
- **变量传递**：将当前循环变量传递给任务
- **循环控制**：支持循环次数限制和条件控制
- **结果收集**：收集所有循环任务的执行结果

## 使用场景

- 当你需要对多个数据项执行相同操作时
- 当你需要批量处理数据时
- 当你需要在工作流中实现循环逻辑时
- 当你需要遍历数据集合并对每个元素执行任务时

## 触发条件

- `loop`
- `iterate`
- `for each`
- `loop over`
- `repeat`

## 依赖关系

- `task-runner` - 用于执行循环中的任务

## 使用方法

### 1. 遍历数据集

```bash
# 使用命令行遍历数据集
lingflow run loop-iterator --params '{"items": [1, 2, 3, 4, 5], "tasks": [{"skill": "process-item", "params": {"item": "${item}"}}]}'
```

### 2. 带循环变量的循环

```bash
# 带循环变量的循环
lingflow run loop-iterator --params '{"items": [{"name": "item1"}, {"name": "item2"}], "tasks": [{"skill": "process-item", "params": {"name": "${item.name}"}}]}'
```

## 最佳实践

1. **明确循环目标**：在使用循环前，明确循环的目标和数据集
2. **合理设置循环**：根据实际需求设置合理的循环次数和条件
3. **变量传递**：合理使用循环变量，传递给任务
4. **错误处理**：处理循环过程中的错误，确保循环可以正常完成
5. **性能优化**：对于大型数据集，考虑分批处理，避免性能问题

## 故障排除

- **数据集为空**：检查数据集是否为空
- **任务执行失败**：检查任务配置是否正确
- **变量传递错误**：检查循环变量的使用是否正确
- **循环次数过多**：检查循环次数是否合理，避免无限循环
- **性能问题**：对于大型数据集，考虑优化处理方式
