# lingflow Quality Gate - GitHub Action

**AI-powered code review and quality gate** for your GitHub workflows

---

## 🚀 快速开始

### 基础用法

```yaml
name: lingflow Quality Gate

on:
  pull_request:
    types: [opened, synchronize, reopened]

jobs:
  lingflow-review:
    runs-on: ubuntu-latest
    name: Code Review
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Run lingflow Review
        uses: lingflow/actions/quality-gate@v1
        with:
          command: review
          path: ./src
          github_token: ${{ secrets.GITHUB_TOKEN }}
```

### 完整配置

```yaml
- name: lingflow Quality Gate
  uses: lingflow/actions/quality-gate@v1
  with:
    # 命令：review, test, optimize, analyze
    command: 'review'

    # 目标路径
    path: './src'

    # GitHub Token（必需，用于 PR 评论）
    github_token: ${{ secrets.GITHUB_TOKEN }}

    # lingflow API Key（可选，高级功能）
    api_key: ${{ secrets.LINGFLOW_API_KEY }}

    # 发现问题是否失败
    fail_on_error: 'false'

    # 输出格式：markdown, json, summary
    output_format: 'markdown'
```

## 📖 可用命令

### 1. 代码审查（review）

```yaml
- uses: lingflow/actions/quality-gate@v1
  with:
    command: review
    path: ./src
```

**审查维度**:
- 复杂度分析
- 重复代码检测
- 安全漏洞扫描
- 代码风格检查
- 文档完整性
- 测试覆盖率
- 性能问题
- 错误处理

### 2. 运行测试（test）

```yaml
- uses: lingflow/actions/quality-gate@v1
  with:
    command: test
    path: ./tests
```

**功能**:
- 运行单元测试
- 生成覆盖率报告
- 失败测试分析

### 3. 代码优化（optimize）

```yaml
- uses: lingflow/actions/quality-gate@v1
  with:
    command: optimize
    path: ./src
```

**优化类型**:
- 结构优化（降低复杂度）
- 性能优化
- 简洁优化（减少重复）

### 4. 静态分析（analyze）

```yaml
- uses: lingflow/actions/quality-gate@v1
  with:
    command: analyze
    path: ./src
```

## 🎯 使用场景

### 1. PR 质量门禁

```yaml
name: Quality Gate

on:
  pull_request:
    branches: [main]

jobs:
  quality-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: lingflow/actions/quality-gate@v1
        with:
          command: review
          path: ./src
          fail_on_error: 'true'  # 发现问题则失败
```

### 2. 定期质量检查

```yaml
name: Weekly Quality Check

on:
  schedule:
    - cron: '0 9 * * 1'  # 每周一上午 9 点

jobs:
  weekly-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: lingflow/actions/quality-gate@v1
        with:
          command: analyze
          path: ./
```

### 3. 多项目检查

```yaml
jobs:
  check-all:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Check Backend
        uses: lingflow/actions/quality-gate@v1
        with:
          command: review
          path: ./backend

      - name: Check Frontend
        uses: lingflow/actions/quality-gate@v1
        with:
          command: review
          path: ./frontend
```

## 🔧 配置选项

### 输入参数

| 参数 | 必需 | 默认值 | 描述 |
|------|------|--------|------|
| `command` | 否 | `review` | lingflow 命令 |
| `path` | 是 | `./src` | 目标路径 |
| `github_token` | 是 | `${{ github.token }}` | GitHub Token |
| `api_key` | 否 | - | lingflow API Key |
| `fail_on_error` | 否 | `false` | 是否失败 |
| `output_format` | 否 | `markdown` | 输出格式 |

### 输出参数

| 参数 | 描述 |
|------|------|
| `result_file` | 结果文件路径 |
| `issues_found` | 发现的问题数量 |
